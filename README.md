# Wonders

## Описание
Адаптивный лендинг для сервиса поиска альпийских автобусных маршрутов.  
Сайт включает форму поиска, счётчик пассажиров, выпадающие списки станций, календарь, бургер-меню для мобильных устройств, блок с вопросами и раскрывающимися ответами.

---

## Этапы разработки
1. **HTML** – каркас сайта.
2. **CSS** – стилизация и адаптивная верстка (1440px, 768px, 360px).
3. **JavaScript** – логика работы формы, бургер-меню, календаря и валидации.

## Функционал
- Поиск маршрутов: поля "Откуда" и "Куда" с автодополнением, значения которых не могут совпадать, в противном случае появляется модальное окно с уведомлением.
- Выбор типа поездки: Round Trip / One Way. Первоначально ни один из видов не активен для того, чтобы проверить, выбрал ли пользователь тип поездки перед отправкой формы.
- Календарь с отображением текущего и следующего месяцев, с ограничениями выбора для обратной даты, она не может быть раньше даты отправления, в противном случае появляется модальное окно с уведомлением. В случае выбора типа поездки One Way поле для выбора обратной даты становится недоступным. При выборе первой даты подсвечивается поле даты отправления, при выборе второй даты подсвечивается поле даты возвращения, а также в календаре выделяются дата отправления и дата возвращения синим цветом, а даты между ними подсвечиваются серым цветом. Кнопка Apply фиксирует значения двух дат, при этом подсвечивание полей с выбранными датами убирается. Если хотя бы одно из полей с выбором дат не заполнено, то при нажатии на Apply появляется модальное окно с уведомлением. Кнопка Reset сбрасывает значения выбранных дат, а также все выделения в календаре. 
- Счётчик пассажиров (максимум 12). Первоначальное значение 0, что говорит о том, что пользователь не указал количество пассажиров. Кнопки + и - в двух состояниях: в активном состоянии голубые, неактивные - серые (кнопка - серого цвета, когда значение счетчика первоначальное 0 или 1, так как количество пассажиров не может быть меньше 1, кнопка + серого цвета, когда значение 12, так как количество не может быть больше 12)
- Бургер-меню для мобильных устройств.
- Кнопка Find My Alpine Ride с переходом на bus-list.html. Настроена валидация. Если хотя бы одно поле не заполнено, появляется модальное окно с уведомлением.
- Блок с вопросами и раскрывающимися ответами.
